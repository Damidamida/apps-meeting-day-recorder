# Правила работы Codex

## Источник контекста

`PROJECT_STATE.md` является единым источником истины о контексте проекта.

- Перед каждой задачей Codex обязан прочитать `AGENTS.md` и `PROJECT_STATE.md`.
- После каждой завершенной задачи Codex обязан обновить `PROJECT_STATE.md` в той же ветке и в том же PR.
- Если `PROJECT_STATE.md` отсутствует, Codex обязан остановиться и запросить уточнение вместо догадок.
- Важные решения нельзя удалять, пока они явно не заменены новым решением.

## Перед изменениями

Codex обязан:

- подтвердить репозиторий;
- подтвердить текущую базовую ветку;
- подтвердить этап задачи;
- подтвердить, что явно исключено из задачи.

## Во время работы

- Сохранять изменения небольшими и удобными для ревью.
- Работать только в отдельной ветке `codex/*`.
- Не пушить напрямую в `main`.
- Не выполнять merge в `main`.
- Не добавлять функции за пределами запрошенного этапа.
- Сохранять local-first поведение.
- Не коммитить секреты, записи, аудио, персональные транскрипты, `config.yaml` и `.env`.
- Сохранять русский язык для всех пользовательских текстов.

## После реализации

В той же ветке и в том же PR обновить `PROJECT_STATE.md`:

- текущий статус;
- выполненную работу;
- следующий шаг;
- риски и решения при необходимости;
- запись changelog с именем ветки и номером PR, если он известен;
- результаты тестов.

## Язык проекта

Рабочий язык проекта — русский.

- Заголовки и описания PR пишутся на русском.
- Текст `README.md` и `PROJECT_STATE.md` пишется на русском.
- Пользовательские тексты UI, ошибки и статусы приложения пишутся на русском.
- Метки экрана ревью, шаблоны итогов, метки задач и заголовки генерируемых Markdown-файлов пишутся на русском.
- Английский допустим для идентификаторов кода, имен классов, функций, модулей и файлов, пакетов, CLI-команд, внешних API и обязательных технических литералов.
