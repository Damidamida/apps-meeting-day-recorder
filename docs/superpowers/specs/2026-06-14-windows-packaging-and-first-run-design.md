# Дизайн Stage 10: Windows-установщик и мастер первого запуска

## Цель

Подготовить BK Scribe к передаче нетехническим коллегам как устанавливаемое Windows-приложение. Пользователь должен получить один понятный установщик, поставить приложение без прав администратора, пройти обязательную настройку и только после успешных проверок начать рабочий день.

Stage 10 делится на два отдельных PR:

1. PR 1: Windows-установщик, упаковка и брендинг BK Scribe.
2. PR 2: обязательный мастер первого запуска и проверки готовности.

Такой раздел нужен, чтобы сборка/установщик не смешивались в одном ревью с UI-мастером, проверками OBS и AI-настройками.

## Согласованные решения

- Финальное пользовательское название приложения: `BK Scribe`.
- Иконка приложения предоставлена пользователем: `C:\Users\damir\OneDrive\Рабочий стол\Новая папка (2)\123\meeting_day_recorder_corporate_icon_v2.ico`.
- В PR 1 иконку нужно скопировать в репозиторий как ассет приложения, например `app/assets/bk_scribe.ico`.
- Название `BK Scribe` используется в левом меню, заголовке окна, `.exe`, установщике, ярлыках, меню Пуск и uninstall-записи Windows.
- Репозиторий, Python-пакеты и внутренние технические идентификаторы не переименовываются без отдельной необходимости.
- Установщик полностью на русском языке.
- Установка выполняется без прав администратора и только для текущего пользователя.
- Установщик не ставит OBS и не пытается автоматически настраивать OBS.
- OBS обязателен для готовности приложения, но проверяется и настраивается через мастер первого запуска.
- AI-итоги обязательны для коллег.
- Транскрипция обязательна для коллег.
- Без успешного мастера первого запуска нельзя начать рабочий день.
- Установщик управляет файлами приложения, но не удаляет и не перезаписывает пользовательские рабочие данные.

## PR 1: Windows-установщик, упаковка и брендинг

### Результат

Пользователь получает `.exe`-установщик BK Scribe, который ставит приложение в пользовательскую папку Windows, создает ярлыки, добавляет uninstall и запускает приложение после установки.

### Рекомендуемый инструмент

Использовать Inno Setup в per-user режиме, потому что он хорошо подходит для:

- `.exe`-установщика;
- русской локализации;
- ярлыков на рабочем столе и в меню Пуск;
- иконки установщика и приложения;
- uninstall-записи Windows;
- установки без прав администратора.

### Поведение установки

- Папка установки приложения выбирается в установщике.
- Путь по умолчанию: `%LOCALAPPDATA%\BK Scribe`.
- Установщик показывает пояснение: файлы приложения и рабочие данные хранятся отдельно.
- Папка рабочих данных не выбирается в установщике; она выбирается в мастере первого запуска.
- После установки показывается чекбокс `Запустить BK Scribe`, включенный по умолчанию.
- Если приложение запускается впервые или настройка не завершена, открывается мастер первого запуска.
- Если это обновление уже настроенного приложения, открывается обычный интерфейс.

### Что устанавливается

- Собранное Windows-приложение BK Scribe.
- Все Python/runtime-зависимости, необходимые для запуска без установленного Python у пользователя.
- Bundled `ffmpeg`, доступный приложению без системного PATH.
- Иконка BK Scribe.
- Ярлык на рабочем столе.
- Ярлык в меню Пуск.
- Uninstall-запись Windows.
- Базовый локальный конфиг без секретов.

### Что не устанавливается

- OBS.
- Драйверы, кодеки и системные службы.
- AI-ключи.
- Пользовательские записи, transcript, summary и рабочие дни.
- Глобальные переменные окружения Windows.

### Обновление версии

Новая версия BK Scribe ставится поверх старой и заменяет только файлы приложения.

Обновление не должно удалять или перезаписывать:

- `config.yaml`;
- локальный `.env` с ключами;
- выбранную папку данных;
- рабочие дни;
- записи;
- `audio.wav`;
- `transcript.md` / `transcript.json`;
- `summary.md` / `00_day_summary.md`;
- legacy summary-файлы.

Uninstall не должен молча удалять рабочие данные. Если uninstall показывает пояснение, текст должен явно говорить, что рабочие данные сохранены в выбранной пользователем папке.

### Брендинг

В PR 1 нужно заменить пользовательское отображение `Meeting Day Recorder` на `BK Scribe`:

- левое меню приложения;
- заголовок окна;
- название `.exe`;
- название установщика;
- название ярлыков;
- меню Пуск;
- uninstall-запись Windows.

Внутренние технические имена можно оставить прежними, если их переименование не требуется для упаковки.

## PR 2: Мастер первого запуска

### Результат

После установки нетехнический пользователь проходит обязательный мастер настройки. Пока мастер не завершен успешно, приложение не дает начать рабочий день.

Мастер можно открыть позже из настроек как `Мастер настройки`.

### Общая модель

Мастер работает как setup-gate:

- при первом запуске открывается автоматически вместо экрана `Рабочий день`;
- при незавершенной обязательной настройке блокирует `Рабочий день`, `Ревью`, `Архив` и плавающую кнопку старта;
- `Настройки` и `Справка` остаются доступными, чтобы пользователь мог вернуться к мастеру или прочитать инструкцию;
- показывает понятные шаги без технического шума;
- сохраняет настройки локально;
- позволяет повторять проверки;
- не создает реальные встречи, записи, transcript или итоги пользователя.

Мастер реализуется как полноэкранный экран внутри основного окна приложения, а не как цепочка отдельных popup-окон. Верхний блок `Готовность к работе` не используется: вместо него достаточно заголовка `Настройка BK Scribe`, короткого пояснения и компактного прогресса. Левая колонка шагов и правая рабочая область должны иметь одинаковую высоту.

Шаги строго последовательные:

- следующий шаг недоступен, пока текущий не имеет статус `Готово`;
- кнопка `Далее` выключена, пока текущий шаг не прошел успешную проверку;
- будущие шаги визуально заблокированы и не открываются кликом;
- пользователь может возвращаться к уже пройденным шагам;
- при изменении поля на пройденном шаге статус этого шага и всех зависимых следующих шагов сбрасывается до новой проверки.

Визуальный референс зафиксирован в `docs/mockups/first-run-wizard-bk-scribe.html`.

### Обязательные шаги

1. `Папка данных`
   - Пользователь выбирает папку для рабочих дней, записей, transcript и итогов.
   - Путь по умолчанию: `Документы\BK Scribe`.
   - Проверка: папка существует или может быть создана, приложение может писать в нее файлы.

2. `OBS`
   - Проверяется OBS WebSocket.
   - OBS должен быть установлен и запущен пользователем.
   - Мастер показывает кнопку `Открыть OBS`, если это возможно.
   - Мастер показывает кнопку `Проверить OBS` / `Повторить проверку`.
   - Если OBS не готов, пользователь видит короткую инструкцию включить WebSocket и повторить проверку.

3. `Аудио`
   - Проверяется bundled `ffmpeg`.
   - Для пользователя статус формулируется как `Аудио из записей: готово`.
   - Этот шаг не должен требовать ручного выбора `ffmpeg`, если bundled binary доступен.

4. `AI Tunnel`
   - Пользователь вводит один `AI Tunnel key`.
   - Этот ключ используется и для транскрипции через AI Tunnel STT, и для AI-итогов.
   - На шаге показывается ссылка `https://aitunnel.ru/` с текстом для получения или оформления ключа.
   - Кнопка `Проверить ключ AI Tunnel` выполняет дешевую проверку доступности ключа.
   - Ключ сохраняется в локальный `.env` или текущий `secrets.env_file` только после успешной проверки.
   - Полный ключ нельзя показывать в logs, metadata, статусах и пользовательских сообщениях.

5. `Транскрипция`
   - Пользователь выбирает обязательный режим транскрипции из выпадающего списка.
   - Первым и выбранным по умолчанию стоит `AI Tunnel STT`.
   - В списке остаются текущие варианты приложения: `AI Tunnel STT`, `faster-whisper`, `Whisper CLI`.
   - Кнопки `Изменить` на этом шаге нет; выбор делается через dropdown.
   - Если выбран `AI Tunnel STT`, проверка транскрипции требует успешно пройденный шаг `AI Tunnel`.
   - Если ключ AI Tunnel еще не проверен, пользователь видит `Сначала проверьте ключ AI Tunnel.`
   - Для коллег транскрипция не может быть пропущена.
   - Проверка не должна создавать реальную встречу.

6. `AI-итоги`
   - Пользователь настраивает обязательную генерацию итогов.
   - Отдельного ввода ключа на этом шаге нет: используется ключ из шага `AI Tunnel`.
   - На шаге выбирается модель итогов из текущих вариантов AI Tunnel.
   - Если ключ AI Tunnel еще не проверен, пользователь видит `Сначала проверьте ключ AI Tunnel.`
   - Проверка итогов отправляет короткий тестовый текст в выбранную summary-модель и ожидает валидный ответ.

7. `Начать работу`
   - Финальный экран показывает все обязательные проверки.
   - Кнопка `Начать работу` активна только когда все проверки успешны.
   - Если есть ошибки, пользователь видит `Продолжить настройку` и список проблем.

Итоговая последовательность: `Папка данных готова` → `OBS готов` → `Аудио готово` → `AI Tunnel key готов` → `Транскрипция готова` → `AI-итоги готовы` → `Начать работу`.

### Критерий готовности

Приложение считается готовым к работе только если:

- папка данных доступна для записи;
- OBS WebSocket подключается;
- `ffmpeg` доступен приложению;
- AI Tunnel key проверен и сохранен локально;
- транскрипция настроена и проверена;
- AI-итоги настроены и проверены.

Если хотя бы один пункт не готов, рабочий день нельзя начать.

### Проверка AI

Проверки должны быть дешевыми и безопасными:

- для итогов отправляется короткий тестовый текст;
- реальные записи пользователя не используются;
- реальные transcript и summary не создаются;
- ошибки показываются простым языком.

Примеры пользовательских статусов:

- `AI-итоги готовы`;
- `Ключ не найден`;
- `Ключ не подошел`;
- `Сервис временно недоступен`;
- `Транскрипция не настроена`;
- `OBS не подключен`.

### Хранение настроек и секретов

- Настройки сохраняются в локальный `config.yaml`.
- Секреты сохраняются в локальный `.env` или текущий поддерживаемый механизм `secrets.env_file`.
- `config.yaml`, `.env`, записи, audio, transcript и summary не коммитятся.
- В logs, metadata и пользовательских сообщениях нельзя показывать полный AI-ключ.

### UX мастера

Мастер должен быть рассчитан на нетехнического пользователя:

- короткие русские заголовки;
- простые статусы `Готово`, `Требует действия`, `Ошибка проверки`;
- одна основная кнопка на шаг;
- подробности ошибки можно показывать вторичной строкой, без stack trace;
- повторная проверка доступна сразу после исправления;
- пользователь понимает, что именно мешает начать работу.

## Исключено из Stage 10

- Автоматическая установка OBS.
- Автоматическая настройка OBS без участия пользователя.
- MSI/корпоративное развертывание через IT.
- Установка для всех пользователей компьютера.
- Админ-установка в `Program Files`.
- Миграция старых пользовательских данных.
- Изменение структуры рабочих дней.
- Переименование репозитория и внутренних Python-пакетов.
- Изменение логики OBS/FFmpeg/STT/Summary pipeline за пределами проверок готовности.
- Удаление пользовательских данных при uninstall.

## Риски и решения

- Корпоративная политика может блокировать запуск per-user приложений из `%LOCALAPPDATA%`.
  - Решение: для текущего scope принять per-user установку; при необходимости позже добавить отдельный IT/admin-вариант.

- OBS может быть не установлен или WebSocket может быть выключен.
  - Решение: не ставить OBS автоматически, а дать понятный шаг проверки и инструкцию в мастере.

- AI-проверка может тратить деньги или зависеть от внешнего сервиса.
  - Решение: использовать короткий тестовый текст и не создавать реальные пользовательские материалы.

- Обновление может случайно затронуть данные пользователя.
  - Решение: жестко разделить папку приложения и папку данных; установщик не управляет папкой данных.

## Приемка PR 1

- Установщик запускается без прав администратора.
- Пользователь может выбрать папку установки приложения.
- По умолчанию приложение ставится в `%LOCALAPPDATA%\BK Scribe`.
- В Windows появляется приложение `BK Scribe`.
- Окно, ярлыки, меню Пуск и uninstall показывают `BK Scribe`.
- Иконка применена к приложению, ярлыкам и установщику.
- `ffmpeg` доступен приложению после установки.
- После установки можно запустить приложение.
- Обновление поверх старой версии не удаляет настройки и данные.
- Uninstall не удаляет рабочую папку данных.

## Приемка PR 2

- При первом запуске открывается мастер первого запуска.
- Без успешного мастера нельзя начать рабочий день.
- Папка данных выбирается в мастере и проверяется на запись.
- OBS обязателен и проверяется через WebSocket.
- Bundled `ffmpeg` проверяется внутри мастера.
- Транскрипция обязательна и проверяется без реальной встречи.
- AI-итоги обязательны и проверяются коротким тестовым текстом.
- Финальная кнопка `Начать работу` активна только при успешных обязательных проверках.
- Мастер можно открыть повторно из настроек.
- Все тексты мастера на русском.

## Проверки реализации

Для PR 1:

- сборка приложения;
- запуск установленного приложения на чистом пользовательском профиле;
- проверка ярлыков, иконок и uninstall;
- ручная проверка обновления поверх предыдущей установки;
- проверка, что рабочая папка данных не удаляется.

Для PR 2:

- unit-тесты проверок готовности;
- UI-тесты сценариев мастера;
- ручная проверка успешной настройки;
- ручная проверка ошибок OBS, AI-ключа и недоступного сервиса;
- smoke-test запуска рабочего дня после успешного мастера.
