# Красивый просмотр итогов и единый файл summary — design-spec

## Контекст

Сейчас приложение использует старую модель ревью:

- для встречи есть `summary_draft.md` и `summary_final.md`;
- для дня есть `00_day_summary_draft.md` и `00_day_summary_final.md`;
- во вкладке `Ревью` пользователь видит Markdown-редактор по умолчанию;
- в `Ревью` есть кнопки `Сохранить черновики`, `Сохранить финальные файлы`, `Открыть папку дня`;
- в `Архиве` есть отдельные действия `Сохранить черновик` и `Сохранить финал`;
- в `Архиве` карточки дня и материалы открываются через отдельные кнопки, а не через саму карточку.

Это больше не соответствует согласованной продуктовой модели. Пользователь хочет видеть итог как готовый материал внутри приложения, а редактировать его только при явном действии.

## Цель

Сделать в `Ревью` и `Архиве` единый UX для итогов:

- по умолчанию показывать красивый просмотр Markdown, а не сырой редактор;
- включать редактирование только по кнопке `Редактировать`;
- сохранять изменения одной кнопкой `Сохранить`;
- убрать разделение на черновики и финальные файлы;
- реально перезаписывать итоговый файл на диске;
- сохранить local-first поведение.

## Новая файловая модель

В приложении остается один итоговый файл для каждого материала:

- итог встречи: `summary.md`;
- итог дня: `00_day_summary.md`.

В пользовательском UI больше нет понятий `черновик`, `финал`, `draft`, `final`.

Старые тестовые данные мигрировать не требуется. Для совместимости реализации можно читать старые файлы как fallback, если новый файл еще отсутствует, но новые записи должны идти только в новую модель:

- встреча сохраняется в `summary.md`;
- день сохраняется в `00_day_summary.md`;
- генерация встречи пишет `summary.md`;
- генерация дня пишет `00_day_summary.md`;
- повторная обработка встречи заменяет `summary.md`;
- обновление итогов дня заменяет `00_day_summary.md`.

## Ревью

### Структура экрана

Текущая структура `Ревью` сохраняется:

- слева список `Встречи за день`;
- справа вкладки материала;
- для встречи есть вкладки `Итоги встречи` и `Транскрипт`;
- для дня первая вкладка показывает `Итоги дня`, вторая вкладка `Транскрипт` показывает ссылки на транскрипты встреч дня.

Меняется содержимое вкладки итогов.

### Режим просмотра

При выборе встречи или `Итоги дня` приложение открывает вкладку итогов в режиме просмотра.

В режиме просмотра:

- Markdown итогов рендерится как структурированный UI;
- заголовки, списки, задачи, риски и обычные абзацы визуально разделены;
- сырой Markdown не показывается как plain text;
- кнопка `Редактировать` находится справа в header открытого материала;
- кнопок `Сохранить`, `Отмена` нет;
- кнопок `Сохранить черновики`, `Сохранить финальные файлы`, `Открыть папку дня` нет.

### Режим редактирования

Кнопка `Редактировать` переключает текущую вкладку итогов в режим редактирования.

В режиме редактирования:

- показывается Markdown-редактор;
- справа в header открытого материала показываются кнопки `Сохранить` и `Отмена`;
- кнопка `Редактировать` скрыта;
- `Сохранить` записывает новый текст в `summary.md` или `00_day_summary.md`;
- после `Сохранить` приложение возвращается в красивый просмотр;
- `Отмена` не меняет файл на диске и возвращает красивый просмотр предыдущего текста.

### Транскрипт

Вкладка `Транскрипт` остается отдельной.

Транскрипт:

- read-only;
- не получает кнопку `Редактировать`;
- не получает кнопку `Сохранить`;
- продолжает открывать ссылки на транскрипты встреч дня внутри приложения.

## Архив

### Структура экрана

Текущая структура `Архива` сохраняется:

- слева список прошлых дней;
- справа материалы выбранного дня;
- сверху поиск и фильтры периода.

Меняется логика карточек материалов справа.

### Список дней слева

Карточка дня слева становится основным действием выбора дня.

Требования:

- отдельная кнопка `Открыть` внутри карточки дня удаляется;
- клик по карточке выбирает день;
- выбранный день визуально отличается от остальных;
- карточка дня остается компактной;
- сегодняшний день не отображается в Архиве.

### Материалы дня справа

Справа показываются карточки:

- `Итоги дня`, если они есть или должны быть доступны для дня;
- встречи выбранного дня.

Карточки работают как аккордеон:

- открыта только одна карточка;
- клик по закрытой карточке открывает ее и закрывает предыдущую;
- закрытая карточка показывает только заголовок, статус и короткую строку состояния;
- открытая карточка показывает красивый просмотр итогов;
- действия находятся справа в header открытой карточки, как в `Ревью`.

### Открытая карточка `Итоги дня`

В режиме просмотра header содержит:

- `Редактировать`;
- `Обновить итоги дня`.

Если прошлый день имеет статус `active`, дополнительно доступно действие:

- `Завершить день`.

`Обновить итоги дня` сохраняет текущую safety-логику: перед запуском показывается предупреждение:

> Если вы вручную меняли Итог дня, обновление заменит ваши изменения.

### Открытая карточка встречи

В режиме просмотра header содержит:

- `Редактировать`;
- `Просмотреть транскрипт`;
- `Повторить обработку`, если встречу можно безопасно повторно обработать.

`Повторить обработку` сохраняет текущую safety-логику: перед запуском показывается предупреждение:

> Если вы вручную меняли Итог встречи, новая обработка заменит ваши изменения.

`Просмотреть транскрипт` переключает содержимое этой же раскрытой карточки на transcript read-only. В этом состоянии действие меняется на `Показать итог`.

### Режим редактирования в Архиве

Для `Итоги дня` и встречи режим редактирования одинаковый:

- кнопка `Редактировать` заменяется на `Сохранить` и `Отмена`;
- остальные рискованные действия скрываются, пока включен редактор;
- `Сохранить` пишет в `00_day_summary.md` или `summary.md`;
- `Отмена` возвращает просмотр без записи на диск;
- после сохранения открытая карточка остается открытой и показывает обновленный красивый просмотр.

## Поиск в Архиве

Поиск продолжает работать локально и не должен заметно тормозить UI.

Требования:

- поиск учитывает `summary.md`, `00_day_summary.md` и transcript;
- fallback-чтение старых файлов допускается только для отображения старых локальных данных;
- результаты поиска под строкой поиска остаются компактными;
- если результат поиска открыт, правая панель выбирает день и раскрывает соответствующий материал;
- если найдено совпадение в transcript, раскрывается карточка встречи в режиме transcript;
- если найдено совпадение в итогах, раскрывается карточка встречи или дня в режиме красивого просмотра итогов.

## Удаляемые и сохраняемые кнопки

### Удалить из `Ревью`

- `Сохранить черновики`;
- `Сохранить финальные файлы`;
- `Открыть папку дня`.

### Удалить из `Архива`

- `Сохранить черновик`;
- `Сохранить финал`;
- кнопка `Открыть` внутри карточки дня.

Кнопка `Открыть` в компактном результате поиска может быть оставлена на переходный период, но предпочтительное поведение — кликабельная строка результата.

### Оставить

- `Редактировать`;
- `Сохранить`;
- `Отмена`;
- `Просмотреть транскрипт`;
- `Показать итог`;
- `Повторить обработку`;
- `Обновить итоги дня`;
- `Завершить день` для прошлого активного дня;
- фильтры `Неделя`, `Месяц`, `Все`.

## Не входит в scope

- История версий итогов.
- Diff ручных изменений.
- Резервные копии перед повторной обработкой.
- Миграция старых тестовых встреч.
- Новый формат transcript.
- Изменение OBS, FFmpeg, STT backend и AI Tunnel настроек.
- Новая календарная визуализация Архива.
- Удаление старых файлов с диска у пользователя.

## Acceptance criteria

- В `Ревью` итог встречи открывается красивым просмотром, а не редактором.
- В `Ревью` итог дня открывается красивым просмотром.
- В `Ревью` вкладка `Транскрипт` остается отдельной и read-only.
- В `Ревью` нет кнопок `Сохранить черновики`, `Сохранить финальные файлы`, `Открыть папку дня`.
- В `Архиве` карточка дня выбирается кликом по карточке, без кнопки `Открыть`.
- В `Архиве` справа открыта только одна карточка материала.
- В `Архиве` открытая карточка показывает красивый просмотр итогов.
- В `Архиве` действия открытой карточки расположены справа в header.
- В режиме редактирования везде есть только `Сохранить` и `Отмена`.
- `Сохранить` пишет в `summary.md` или `00_day_summary.md`.
- Новая генерация итогов встречи пишет `summary.md`.
- Новая генерация итогов дня пишет `00_day_summary.md`.
- Повторная обработка и обновление итогов дня показывают утвержденные предупреждения о перезаписи ручных изменений.
- Поиск Архива учитывает новые итоговые файлы.
- В пользовательском UI нет слов `черновик`, `финал`, `draft`, `final` применительно к итогам.
- Все пользовательские тексты остаются на русском.
- Local-first ограничение сохраняется: аудио, видео, transcript и summary не коммитятся и не отправляются никуда сверх существующих pipeline-правил.

## Проверка spec

- Плейсхолдеров и незаполненных решений нет.
- Scope ограничен `Ревью`, `Архивом`, хранением итогов, генерацией итогов и связанными тестами.
- Требования не переводят этап 8 в `Готово`; после реализации задача должна быть `На проверке`.
