# Meeting Day Recorder — состояние проекта

## 1. Цель продукта

Локальное Windows-приложение для ручной фиксации ad hoc-созвонов в течение рабочего дня, последующей транскрипции, подготовки итогов по каждой встрече и общих итогов дня, ревью пользователем и сохранения финальных файлов локально.

## 2. Основной сценарий

Пользователь начинает рабочий день, вручную запускает и завершает встречи, которые часто появляются на лету из чатов, завершает рабочий день, проверяет черновики итогов, редактирует их и сохраняет финальные файлы.

## 3. Важные ограничения

- В MVP нет интеграции с почтой, календарем и мессенджерами.
- Встречи часто формируются ad hoc из чатов.
- Основной сценарий MVP — ручные кнопки `Start workday`, `Start meeting`, `End meeting`, `End workday`.
- Хранение данных остается local-first.
- Нельзя коммитить записи, аудио, транскрипты с персональными данными, `config.yaml`, `.env` и секреты.
- OBS, ffmpeg, OpenAI API и AI-суммаризация добавляются только на соответствующих этапах.
- Нельзя пушить напрямую в `main`.
- Работа идет через отдельные ветки `codex/*` и PR.
- Весь пользовательский текст должен быть на русском языке.

- После завершенной задачи Codex обязан обновить `PROJECT_STATE.md` в той же ветке и том же PR.

- Если `PROJECT_STATE.md` отсутствует или противоречит задаче, Codex обязан остановиться и запросить уточнение.

## 14. Известные риски и решения на пересмотр

- Восстановление состояния после перезапуска должно сохраняться.
- Выходные файлы должны оставаться вне git.
- Нужно принять решение о промежуточной папке `meetings/`.
- Настройка OBS-сцены и аудио будет ручной предпосылкой.
- Пароль OBS WebSocket хранится только в локальном `config.yaml`, который не добавляется в git.
- Путь записи настраивается в OBS; приложение сохраняет `recording_path`, только если OBS возвращает его после остановки записи.
- Большие записи могут потребовать политику очистки.
- Длинное аудио может потребовать разбиение на части.
- Качество prompt для итогов нужно будет улучшать итерационно.
- Нельзя смешивать этапы и добавлять AI или OBS раньше плана.
- Пользовательские тексты UI и генерируемых Markdown-заглушек переведены на русский язык на этапе Review UI.
